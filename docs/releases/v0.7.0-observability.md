# v0.7.0 — Observability Control Center

**Release Date:** 2026-07-05

## Overview

Sprint 7 introduces a read-only Observability Control Center that inspects Prometheus, Grafana, Loki, Tempo, and OpenTelemetry services via their REST APIs. No mutation of monitoring systems, no dashboard pushes, no alert creation — only inspect and visualize.

## Backend: 13 API Endpoints

| Endpoint | Description |
|---|---|
| `GET /api/observability/health` | Aggregate health of all observability services |
| `GET /api/observability/overview` | Summary counts (alerts, targets, services, dashboards) |
| `GET /api/prometheus/metrics` | Prometheus metric names/values |
| `GET /api/prometheus/targets` | Active scrape targets with health |
| `GET /api/prometheus/rules` | Recording/alerting rules |
| `GET /api/prometheus/alerts` | Active alerts |
| `GET /api/grafana/health` | Grafana database/version info |
| `GET /api/grafana/dashboards` | Dashboard search results |
| `GET /api/grafana/datasources` | Configured datasources |
| `GET /api/loki/logs` | LogQL query results |
| `GET /api/loki/labels` | Label names and values |
| `GET /api/tempo/traces` | Trace search results |
| `GET /api/tempo/services` | Service graph summary |
| `GET /api/otel/collectors` | OTel collector health |

## Backend Architecture

- **Module:** `dashboard/apps/api/src/observability/`
- **Pattern:** Plain sync functions returning `dict`, called from async FastAPI routes (matching terraform/helm modules)
- **HTTP Client:** Lazy singleton `httpx.Client()` in `utils.py` with graceful `_make_request()` that returns `None` on failure
- **Services:** 1 file per system (`prometheus_service.py`, `grafana_service.py`, `loki_service.py`, `tempo_service.py`, `otel_service.py`) + `health_service.py` for aggregate views
- **Schemas:** Pydantic models in `schemas.py` for all 13 endpoints
- **Router:** 13 endpoints in `router.py`, no authentication guards (read-only)
- **Settings:** Reuses existing `PROMETHEUS_URL`, `LOKI_URL`, `TEMPO_URL`, `GRAFANA_URL` from `settings.py`

## Known Limitations

- OTel collector endpoint hardcoded to `http://localhost:4318` (no settings entry)
- Loki `query_logs` uses `query_range` API only (no tailing)
- Tempo `search_traces` uses v1 search API
- AI panel is placeholder/rule-based (no ML integration)
- No authentication on observability endpoints (read-only)
